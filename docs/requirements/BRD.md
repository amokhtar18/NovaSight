## 1. Executive Summary

NovaSight is a multi-tenant SaaS web application designed to democratize data analytics through a low-code/no-code graphical user interface. The platform enables users to ingest, transform, model, and visualize data without requiring programming expertise, while maintaining enterprise-grade security and governance.

### Key Differentiators

- **Template-Based Architecture**: All code generation follows secure, pre-approved templates—eliminating arbitrary code execution risks
- **Complete Self-Service**: From data ingestion to dashboard creation, users control the entire data lifecycle
- **AI-Powered Insights**: Local LLM integration for natural language data exploration
- **Enterprise Multi-Tenancy**: Complete tenant isolation with RBAC and RLS

### 2.3 Core Architectural Constraint

> **THE TEMPLATE ENGINE RULE**
> 
> The system MUST NOT generate arbitrary code from user input or LLM responses. All executable artifacts (DAGs, PySpark jobs, dbt models) MUST be generated by filling pre-approved, security-audited templates with validated user inputs.

### EPIC 1: Data Source Connection Management

**Epic Description:** Enable users to establish and manage connections to external data sources through a secure, guided GUI experience.

**Business Value:** Eliminates the need for manual database connection configuration, reducing setup time from hours to minutes.

---

#### US-1.1: Create Database Connection

**As a** Data Engineer  
**I want to** configure a new database connection through a GUI form  
**So that** I can access external data without writing connection code

**Acceptance Criteria:**

```gherkin
GIVEN I am on the "Data Sources" page
WHEN I click "Add New Connection"
THEN I see a form with fields for:
  - Connection Name (required, unique per tenant)
  - Database Type (dropdown: PostgreSQL, Oracle, SQL Server)
  - Host/IP Address
  - Port
  - Database Name
  - Username
  - Password (masked)
  - SSL Mode (optional)

GIVEN I have filled all required fields
WHEN I click "Test Connection"
THEN the system attempts to connect using provided credentials
AND displays success/failure message with details

GIVEN the connection test succeeded
WHEN I click "Save Connection"
THEN the connection metadata is stored in the tenant's schema
AND credentials are encrypted at rest
AND the connection appears in my connections list
```

**Technical Notes:**
- Credentials encrypted using AES-256 with tenant-specific keys
- Connection metadata stored in `tenant_schema.data_connections` table
- Connection pooling managed at platform level

---

#### US-1.2: Edit Existing Connection

**As a** Data Engineer  
**I want to** modify an existing database connection  
**So that** I can update credentials or connection parameters

**Acceptance Criteria:**

```gherkin
GIVEN I am viewing my connections list
WHEN I click "Edit" on a connection
THEN the form pre-populates with existing values
AND password field shows placeholder (not actual password)

GIVEN I modify any field
WHEN I click "Test Connection"
THEN the system tests with new values without saving

GIVEN the test succeeds
WHEN I click "Save Changes"
THEN the connection is updated
AND an audit log entry is created
```

---

#### US-1.3: Delete Database Connection

**As a** Data Engineer  
**I want to** remove a database connection  
**So that** unused connections don't clutter my workspace

**Acceptance Criteria:**

```gherkin
GIVEN I select a connection to delete
WHEN the connection is used by active ingestion jobs
THEN I see a warning listing dependent jobs
AND must confirm deletion

GIVEN I confirm deletion
WHEN the connection is removed
THEN all associated credentials are securely deleted
AND dependent jobs are marked as "Connection Missing"
AND an audit log entry is created
```

---

#### US-1.4: Browse Source Schema

**As a** Data Engineer  
**I want to** browse tables and columns from a connected source  
**So that** I can understand the source data structure

**Acceptance Criteria:**

```gherkin
GIVEN I have a valid connection
WHEN I click "Browse Schema"
THEN I see a tree view of:
  - Schemas (if applicable)
  - Tables
  - Views

WHEN I expand a table
THEN I see columns with:
  - Column name
  - Data type
  - Nullable flag
  - Primary key indicator

WHEN I select multiple tables
THEN I can proceed to configure ingestion jobs
```

---

#### US-1.5: Connection Health Monitoring

**As a** Data Engineer  
**I want to** see the health status of my connections  
**So that** I can proactively address connectivity issues

**Acceptance Criteria:**

```gherkin
GIVEN I am on the Data Sources page
WHEN I view the connections list
THEN each connection shows a health indicator:
  - Green: Last check successful
  - Yellow: Slow response (>5 seconds)
  - Red: Connection failed
  - Gray: Never tested

GIVEN a connection shows Red status
WHEN I hover over the indicator
THEN I see the error message and timestamp

GIVEN connections are configured
WHEN the system runs hourly health checks
THEN status indicators update automatically
```

### EPIC 2: PySpark Ingestion Job Configuration

**Epic Description:** Provide a low-code interface for configuring PySpark-based data ingestion jobs, including table selection, column mapping, write modes, and SCD handling.

**Business Value:** Enables data teams to build production-grade ingestion pipelines without writing Spark code, reducing development time by 80%.

---

#### US-2.1: Create New Ingestion Job

**As a** Data Engineer  
**I want to** create a new data ingestion job through a wizard  
**So that** I can load data from source to ClickHouse without coding

**Acceptance Criteria:**

```gherkin
GIVEN I have at least one valid data connection
WHEN I click "Create Ingestion Job"
THEN I enter a multi-step wizard:

STEP 1 - Source Selection:
  - Select source connection (dropdown)
  - Browse and select source table(s)
  - Preview sample data (first 100 rows)

STEP 2 - Column Configuration:
  - View all source columns with data types
  - Select/deselect columns to include
  - Rename target column names
  - Apply basic transformations (trim, upper, lower, cast)

STEP 3 - Target Configuration:
  - Specify target schema (defaults to tenant schema)
  - Specify target table name
  - Map source-to-target data types
  - Define primary key column(s)

STEP 4 - Load Settings:
  - Select write mode: Overwrite | Append | Merge
  - If Merge: specify merge keys
  - Configure SCD Type (None, Type 1, Type 2)
  - Set batch size (rows per partition)

STEP 5 - Review & Create:
  - Summary of all configurations
  - Preview generated job (template-filled)
  - Save job configuration
```

**Technical Notes:**
- All configurations stored in `tenant_schema.ingestion_jobs` metadata table
- PySpark job generated from `spark_ingestion_template.py.j2`
- No user input directly executed—all values parameterized

---

#### US-2.2: Configure SCD Type 2 Handling

**As a** Data Engineer  
**I want to** configure Slowly Changing Dimension Type 2 tracking  
**So that** I can maintain historical records of dimension changes

**Acceptance Criteria:**

```gherkin
GIVEN I am configuring an ingestion job
WHEN I select "SCD Type 2" as the change tracking method
THEN additional fields appear:
  - Effective Date Column Name (default: effective_from)
  - End Date Column Name (default: effective_to)
  - Current Flag Column Name (default: is_current)
  - Columns to Track for Changes (multi-select)

GIVEN I complete SCD2 configuration
WHEN the job executes
THEN existing records matching on PK with changed tracked columns:
  - Have their end_date set to job run timestamp
  - Have is_current set to FALSE
AND new records are inserted with:
  - effective_from = job run timestamp
  - effective_to = NULL (or 9999-12-31)
  - is_current = TRUE
```

---

#### US-2.3: Configure Incremental Load

**As a** Data Engineer  
**I want to** configure incremental data loading  
**So that** I only process new or changed records

**Acceptance Criteria:**

```gherkin
GIVEN I am configuring load settings
WHEN I select "Incremental" mode
THEN I must specify:
  - Watermark column (timestamp or integer)
  - Initial watermark value (for first run)

GIVEN incremental job executes
WHEN source is queried
THEN only records WHERE watermark_column > last_watermark are fetched
AND new watermark is persisted after successful load
```

---

#### US-2.4: Job Configuration Versioning

**As a** Data Engineer  
**I want to** see version history of job configurations  
**So that** I can track changes and rollback if needed

**Acceptance Criteria:**

```gherkin
GIVEN I am viewing a job's details
WHEN I click "Version History"
THEN I see a list of all configuration versions with:
  - Version number
  - Timestamp
  - User who made changes
  - Summary of changes

WHEN I select a previous version
THEN I can view full configuration details
AND click "Restore This Version" to rollback
```

---

#### US-2.5: Bulk Column Mapping

**As a** Data Engineer  
**I want to** apply transformations to multiple columns at once  
**So that** I can efficiently configure large tables

**Acceptance Criteria:**

```gherkin
GIVEN I am in the column configuration step
WHEN I select multiple columns using checkboxes
THEN I can apply bulk actions:
  - Include/Exclude all selected
  - Apply transformation to all (e.g., TRIM)
  - Set naming convention (snake_case, camelCase)
  - Set nullable flag

WHEN I apply a bulk action
THEN all selected columns update simultaneously
AND I see a success notification
```
